### Fixed
- **T21 (P0, safety)**: bundled ONNX model outputs raw logits (no Softmax node) but `classifier.dart` compared them against probability-scale thresholds (0.55/0.85), risking the crosswalk-deviation alert silently misfiring or never firing. Fixed by applying a numerically-stable softmax to the raw logits before thresholding ‚Äî `crosswalk_app/lib/services/classifier.dart:188-194` (commit `33786d7`). Verified at unit-test level only (T9); field/real-camera detection-rate re-validation remains open (`docs/Tasks.md` T1/T12).
- **T23 (P0, concurrency)**: `_initCamera()` had no re-entrancy guard, allowing concurrent `CameraController`/ONNX session creation via rapid resume/retry (from `didChangeAppLifecycleState` or the retry button). Fixed with an `_isInitializing` guard wrapping the whole body in try/finally, and the retry button is now disabled while a call is in flight ‚Äî `crosswalk_app/lib/screens/camera_screen.dart:54-55,135-137,273` (commit `a9b77f4`).
- **T24 (P1, memory leak)**: `Classifier.init()` unconditionally created a new `OrtSession` without releasing the previous one, and re-initialized the non-idempotent native `OrtEnv` on every call. Fixed by releasing the prior session before reassigning and gating `OrtEnv.instance.init()` to once per instance ‚Äî `crosswalk_app/lib/services/classifier.dart:44-53,221-226` (commit `bc0bba8`).
- **T25 (P1, stale safety cue)**: `FeedbackService.alert()` did not stop prior in-progress TTS speech before speaking a new alert. Load-bearing on iOS, where `AVSpeechSynthesizer` queues utterances by default (Android's `flutter_tts` 4.2.5 already defaults to `QUEUE_FLUSH`). Fixed with `await _tts.stop()` before `speak()` ‚Äî `crosswalk_app/lib/services/feedback_service.dart:42-43` (commit `e5ca05f`).
- **T29 (P2, iOS-only latent bug)**: the BGRA8888 (iOS single-plane) branch of `Classifier._preprocessCamera` passed the raw plane `.buffer` to `img.Image.fromBytes` with no `bytesOffset`/`rowStride`, which would misalign every pixel whenever the plane's `Uint8List` view has a non-zero `offsetInBytes` or the native row stride exceeds `width * 4` (row padding) ‚Äî both allowed by the `camera` plugin's platform interface. Currently dead code (the app forces `ImageFormatGroup.yuv420`, which iOS does not support, per `camera_screen.dart`), but fixed ahead of iOS support by passing `bytesOffset`/`rowStride` through explicitly. Fixed logic extracted into a new `@visibleForTesting` `Classifier.convertBGRA8888()` method (behavior-preserving) and covered by 3 new unit tests that hand-construct a `CameraImage`/`CameraImagePlane` via `CameraImage.fromPlatformInterface` to exercise non-zero offset, row padding, and both combined ‚Äî `crosswalk_app/lib/services/classifier.dart`, `crosswalk_app/test/classifier_test.dart` (`docs/Tasks.md` T29).
